語音識別系統實測報告

真實截圖佐證與功能深度比對

System A (Maxnion) vs. System B (BC 系統)

文件來源：實際測試錄音檔與系統匯出截圖

1. 執行摘要 (Executive Summary)

System A (Maxnion)

定位：全自動 AI 代理人

  • 高準確：自動修正關鍵字錯誤。
  • 高效率：自動產出結構化報告。
  • 風險：黑箱誤導 (AI 幻覺)。

System B (BC 系統)

定位：輔助型編輯工具

  • 高透明：保留原始時間軸。
  • 低效率：需大量人工聽打。
  • 用途：適合法務/字幕。

2. 介面功能設計佐證

真實系統介面截圖：自動化流水線 vs 人工編輯器

System A (Maxnion) 任務列表
設計邏輯：
介面顯示「步驟 1, 2, 3, 4」的任務清單，由不同的 AI 幫手（如：訪談摘要助手）自動執行，使用者無需操作編輯器。
System B (BC 系統) 編輯器
設計邏輯：
介面核心是「播放進度條」與每一句的「時間碼 (00:00:32)」，設計目的是為了讓人類聽打員進行逐字校對。

System A 風險：黑箱誤導

推薦方案：SOP 控管
